# MindSpeed MM FAQs

| No. | Issue Summary | Impact | Current Solution | Future Plan | Reference |
|:------:|:------:|:------:|:------:|:------:|:------:|
| 1 | CANN version mismatch causing training errors | Training fails to start or operator errors occur. | Ensure strict version compatibility among driver/firmware, CANN Toolkit/Kernels/NNAL, PyTorch, and torch_npu. Refer to [Version Compatibility Notes](./release_notes_mm.md#version-compatibility-notes). | Continuously update the compatible versions. | [Installation Guide](pytorch/install_guide.md) |
| 2 | `ModuleNotFoundError: No module named 'acl'` | Unable to import Ascend-related modules | Check if `source /usr/local/Ascend/ascend-toolkit/set_env.sh` has been executed. It is recommended to add it to `~/.bashrc` | - | [Installation Guide](pytorch/install_guide.md) |
| 3 | Incorrect installation order of the CANN NNAL package causing `libatb.so` not found | Dynamic library missing error during inference or training | The NNAL package must be installed after executing `source /usr/local/Ascend/ascend-toolkit/set_env.sh`; otherwise, the dependency path cannot be found. | - | [Installation Guide](pytorch/install_guide.md) |
| 4 | Script hangs without error when launching multi-node multi-card training | Unable to start distributed training | Ensure `pdsh` is installed; otherwise, the multi-node multi-card launch script will hang. Install via `apt install pdsh` or `yum install pdsh`. | - | - |
| 5 | HuggingFace weights cannot be used directly for training | Training fails to start. | HuggingFace weight formats (safetensors/bin) cannot be directly used for Megatron training. Use the `mm-convert` tool to convert them to MindSpeed MM format | - | [Weight Conversion](./features/mm_convert.md) |
| 6 | TP/PP parameters during weight conversion inconsistent with training | Weight loading fails during training. | The `tp_size`, `pp_size`, `llm_pp_layers`, and `vit_pp_layers` used during weight conversion must be consistent with the parallel configuration in the training script.| - | [Weight Conversion](./features/mm_convert.md) |
| 7 | `tp_size` exceeding `num_key_value_heads` | Tensor parallel sharding fails. |  `tp_size` cannot exceed `num_key_value_heads` in the model configuration; otherwise, KV heads cannot be evenly distributed across cards. | - | - |
| 8 | Number of weight files after training differs from the original HF model | Incomplete weight | Training generates Megatron-Core (mcore) format weights, which are converted to standard HF format via `mg2hf`. The number of files varies due to different sharding strategies, but the parameters are complete. The `model.safetensors.index.json` needs to be regenerated. | - | [Weight Conversion](./features/mm_convert.md) |
| 9 | NPU Out of Memory (OOM) | Training interrupted. | Take the following measures: 1. Reduce `micro-batch-size` (minimum value: 1); 2. Increase TP/PP parallel degree (TP  × PP ≤ number of NPUs); 3. Reduce `seq-length`; 4. Enable recomputation (`--recompute-granularity full --recompute-method block --recompute-num-layers`); 5. Use ChunkLoss to reduce peak memory. | - | [ChunkLoss](./features/chunkloss.md) |
| 10 | `learning_rate=None` assertion error when LLM PP is sharded to 0 layers | Training crashes immediately after startup. | The LLM part in the PP sharding configuration cannot have a stage with 0 layers. For example, `llm=[0,8,10,10]` causes this issue. Adjust to `llm=[1,7,10,10]` to ensure each stage has LLM layers. | Optimize the learning rate loading logic to support 0-layer stages. | - |
| 11 | Data preprocessing timeout or synchronization error | Training cannot start with large datasets | Increase `--distributed-timeout-minutes`. For very large datasets (millions of samples or more), preprocess in batches or use higher-performance storage. | Optimize data preprocessing parallel efficiency. | - |
| 12 | Incorrect NIC name causing communication timeout | Multi-node training cannot start. | Use `ifconfig` to check the NIC name and set the corresponding environment variables: `export HCCL_SOCKET_IFNAME=<NIC name>`, `export TP_SOCKET_IFNAME=<NIC name>`, `export GLOO_SOCKET_IFNAME=<NIC name>`. | - | - |
| 13 | Timeout error when saving checkpoints | Weight saving fails after training. | Ensure normal disk I/O bandwidth. A single node's maximum file size of approximately 60G must be saved within 36 minutes. This error can also be ignored as it does not affect the saved weights. | - | - |
| 14 | `syntax error near unexpected token` after a script copy | Script cannot be executed. | Line endings mismatch when a script is copied from Windows to Linux. Execute `dos2unix xxx.sh` or set `:set ff=unix` in vim | - | - |
| 15 | Environment variable conflicts between different CANN versions | Training anomalies in a multi-version environment | It is recommended to use Docker to isolate different versions of CANN packages to prevent environment variable interference. | - | - |
| 16 | Unbalanced load between fast and slow devices during multimodal model training | Low training efficiency | Use multimodal heterogeneous PP sharding to configure different PP layer distributions for the visual encoder, audio encoder, and LLM, avoiding excessive load on certain devices. | - | [Heterogeneous Parallelism](./features/hetero_parallel.md) |
| 17 | Training loss not converged | Poor model performance | Check: 1. Data quality and cleaning logic; 2. Learning rate and warmup strategy; 3. Whether pretrained weights are used for initialization; 4. Whether data paths and image paths are correct. | - | - |
| 18 | Recommended key environment variable configurations | Impact on training performance and stability. | Recommended configurations: `export ASCEND_GLOBAL_LOG_LEVEL=1`, `export TASK_QUEUE_ENABLE=2`, `export CPU_AFFINITY_CONF=2`, `export HCCL_CONNECT_TIMEOUT=600`, and `export PYTORCH_NPU_ALLOC_CONF=expandable_segments:True`. | - | - |
| 19 | Megatron-LM version mismatching with MindSpeed MM | Installation or runtime errors | MindSpeed MM requires a compatible version of Megatron-LM. Please refer to the specified commit or tag in the [Installation Guide](pytorch/install_guide.md) for checkout. | - | [Installation Guide](pytorch/install_guide.md) |
| 20 | Missing model-specific dependencies in the Docker image | Model training errors | The Docker image only contains pre-installed basic dependencies like torch, torch_npu, and decord. Additional dependencies must be manually installed in the base environment according to the target model's README. | - | - |
